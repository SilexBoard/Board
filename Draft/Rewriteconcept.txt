Allgemeines Konzept für das holen von Informationen und die Ausgabe der Seite (Frontend).

* Grundinformationen holen / Initialisieren
	* XML-Wrapper
	* Datenbankverbindung
		* MySQL(i)
		* SQLite
		* PGSQL (PostgreSQL)
	* Login / Registrierung / Abmeldung
		* Wurde sich eingeloggt / registriert / abgemeldet?
	* Sessions
		* Wenn jetzt eingeloggt, alte Session löschen und neue mit UserID, etc erstellen.
	* (Globale) Konfiguration
		* Konfiguration aus der Datenbank lesen.
	* "Globale" Formulare (Können (fast) auf jeder Seite abgeschickt werden)
		* Wurde die Sprache geändert?
		* Wurde der Style geändert?

* Benutzer Informationen holen
	* Benutzerrechte
		* Gruppenreche aus der Datenbank lesen
		* Gegebenfalls mit Benutzerrechten (aus der Datenbank) ergänzen und (teilweise) überschreiben	
	* Benutzerinfos
		* In der Userclass speichern
		* Benutzername
			* To Template -> {{ User.Name }}
		* Avatar
			* To Template -> {{ User.Avatar }}
	* Sprache festlegen
		* Bei Registrierten Benutzern aus der Datenbank lesen
		* Bei Gästen aus der Session/dem Cookie lesen
		* Falls in $_GET angeben, diese Sprache verwenden.
		* To Template -> {{ Language }} array [Titel (in der aktuellen Sprache), Shortcut, ...]
	* Style festlegen
		* To Template -> {{ Style }} array [Titel, Path, ...]
	* Auf welcher Seite befindet sich der User?
		* In der Page-Klasse speichern.
		* To Template -> {{ Page }} array [Seitentitel, Name, ...]

* Vorbereiten der Ausgabe
	* Datenbankverbindung schließen
	* Session beenden?!
* Ausgabe
	* Template Variablen setzen
		* Sprachvariablen setzen (Aus Languageclass)
		* Weitere benötigte Variablen
	* Templates Kompilieren
	* Kompilierte Templates ausgeben
	* Schluss! (Nach der Ausgabe erfolgt keinerlei PHP-Code mehr! (Außer Debugging während der Entwicklung -> Siteload))
